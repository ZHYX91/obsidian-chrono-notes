# UI/UX 规范

## 1. 总体原则

- 优先适配 Obsidian 侧边栏的窄宽度，也能在主区域扩展。
- 遵循 Obsidian 主题变量，不维护与主题竞争的独立设计系统。
- 信息逐级展示，避免把所有节假日、ICS 和统计文本同时塞入日期格。
- 颜色必须与形状、文本或无障碍标签共同表达状态。
- 桌面鼠标、键盘、触摸和触控板都应有完整路径。

## 2. 顶部导航

主日历标题行把上一周期、当前周期入口和下一周期组成左侧紧凑导航组：方向按钮与周期入口之间固定为 `6px`，同一周期入口内部的年份/月份或周历年/周号之间固定为 `2px`。“今天”或“本月”固定在右侧，两组之间使用可伸缩空白并至少保留 `12px`。月视图显示独立年份与月份入口，周视图显示独立周历年与周号入口，年视图只显示年份入口；月/周使用“今天”，年使用“本月”，后者回到当前年并以月粒度选择当前月。周入口不再打开迷你日历，选择周历年或周号后保持周视图，并保留当前所选日期在一周内的星期位置；目标周年没有第 53 周时收敛到最后一个有效周。视图切换位于次级工具行。月网格始终只有一个可见日期进入 roving Tab 顺序，日以外的周/月选择或跨查询选择不能清空该停靠点；热力模式隐藏的外月格不参与。方向键在当前月份边界周网格内移动焦点，跨月时同步切换显示月份；周序号获得焦点时，ArrowRight 进入该周第一个可用日期并同步日粒度选择。

已识别的周期 Markdown 笔记在原生视图头下显示紧凑 Note Navbar：上一周期、带日历图标的周期标签、下一周期、打开 Chrono Notes，以及下一个可实际打开的上级周期。前后/选日与可选上级周期组成整体居中的导航组，上级操作与同级导航保留清晰间距；打开日历单独位于末端动作区。点击周期标签打开同一迷你日历，选择后打开当前类型的周期笔记。相邻上级类型未启用、pattern 为空或无法通过生成路径再反向识别时，按钮继续指向再上一级有效类型；没有有效上级类型时不显示。窄编辑器只隐藏上级周期文字，保留带完整 ARIA/title 的图标按钮。Navbar 归属于每个打开的周期 Markdown 叶子；焦点转到侧边栏时不得卸载，分栏中的周期笔记各自持有独立实例。移动端按“系统安全区—原生视图头—Navbar—正文”排列：Navbar 位于正文滚动区外并占据真实布局高度，phone 浮动导航或自动全屏只补一次宿主头部占位，普通手机、平板和桌面不得产生额外空带；相关区间展开只压缩正文的可滚动高度，不得覆盖原生按钮或正文。周记/月记下方可折叠显示最多五条与周期边界相交的区间笔记，单击在新标签打开，超出显示 `+N`；折叠状态持久化。普通笔记和禁用/部分路径匹配不挂载 Navbar。

命令面板同时提供迷你日历和直接跳转。直接跳转保持旧版 `yyyy-MM-dd`、`yyyy/MM/dd`、`yyyy.MM.dd` 与 `yyyyMMdd` 输入兼容，非法日期不得自动滚动到相邻月份；成功后统一打开主月视图并选择目标日期。

迷你日历沿用同一布局语法：上一月、基准宽度 `132px` 且可在窄宽收缩的年月入口和下一月组成左侧 `6px` 间距的紧凑导航组，“今天”固定在右侧，两组之间的可伸缩空白至少为 `12px`；底部不再重复“今天”。日期模型仍只提供与目标月相交的 4–6 个完整边界周，呈现层用六个固定高度行轨道保留稳定的弹窗尺寸，不生成、显示或暴露完全无关的占位周。月份变化不得移动标题行控件。

```text
主日历·月 [‹] [2026⌄][9月⌄] [›] ──────── [今天]
主日历·周 [‹] [2026⌄][第40周⌄] [›] ───── [今天]
主日历·年 [‹] [2026⌄] [›] ───────────── [本月]
迷你日历  [‹] [2026 年 9 月⌄] [›] ────── [今天]
```

热力图和视图相关操作放在次级工具区，不与日期导航竞争。月/年热力图各自保留独立开关，但启用后共用同一个统计维度、阈值、0–4 级图例和无障碍语义；关闭时只显示 flame 开关。周视图不显示热力图控件或禁用占位。窄宽先隐藏图例；仍放不下时，完整的维度与 flame 工具组换到下一行并右对齐，不缩小文字或拆散控件。

次级工具区与主内容之间按视图保留不同的视觉节奏。月视图维持工具区底部 `4px`、`24px` 星期表头及表头到日期网格 `6px` 的既有结构。周视图与年视图没有星期表头，因此从完整工具区（包括窄宽换行后的第二行）底边到首个主内容格或区块保留 `8px` 有效间距。不得用空白或伪造的星期表头补足高度，也不得为实现该间距改变日期格、周期入口或热力格的内部 padding、gap 或高度。

标题日期入口必须支持高效选择目标年、月、周历年和周，并为公历年/月入口提供打开对应年记、季度笔记或月记的动作。可以使用可访问的 Popover、Modal 或分层选择器替代旧版固定 20 年/季度表格，但键盘、新标签打开和同一时刻只显示一个选择界面的语义必须保留。周历年遵循现有 ISO 周号语义；一年动态包含 52 或 53 周，不能硬编码。

月热力图使用 flame 图标按钮和 `aria-pressed` 表达模式。启用后同时显示共享统计维度和与月格强调色一致的 0–4 级图例；日期格仅保留水平和垂直居中的日期号与四档主题强调色，不渲染区间甘特，跨月格保持网格占位但隐藏内容并退出交互。维度、原始值和等级必须进入日期格无障碍名称与悬浮/焦点预览。热力等级始终拥有背景表达优先权；今日叠加贴格子内缘的中性 `1px` 内边框，选择叠加位于格子边界之外的强调色 `2px` 外框，焦点再位于 selected 外侧作为独立最外层。三种状态均不得覆盖数据背景，也不改变文字颜色、字重或装饰。切回普通模式后恢复原有选择、打开与日期元数据表达。

## 3. 月视图日期格

显示优先级：

1. 顶部左侧的笔记状态/任务进度、顶部右侧的地区标记，以及下一行居中的日期号；
2. 一个或两个历法扩展；
3. 地区节假日；
4. ICS 摘要。

顶部附件行占满日期格的可用宽度。状态圆点与任务进度共用固定左锚点，地区标记使用固定右锚点；有任务时进度只从左锚点向右扩展。两端至少保留 `4px` 间距，窄格优先缩短进度条，不能移动任一锚点或发生重叠。当 `8px` 状态与必要间距仍不足以容纳多字地区标记时，只允许省略标记文字，完整语义继续保留在 tooltip 和无障碍名称中。

日期标题固定为两层：第一层左侧显示状态圆点或横向任务进度，右侧显示至多一个地区标记；第二层由日期号独占整行并几何居中，日期号与后续详情之间保持 `3px` 有效间距。第一层即使两侧为空也保留稳定高度，日期号上边界必须低于两侧内容下边界，不能依赖绝对 `top`、平移日期或按组合打补丁。地区标记使用无外框、透明背景的紧凑文字。日期格其余信息使用纵向内容轨道，不用绝对定位 footer 交换视觉顺序。单个历法扩展居中；两个扩展在日期格约 `60px` 以上时等宽并排，在窄日历中自动纵向排列。单选与双选使用相同字号，不隐藏、不缩小、不横向滚动。每个历法同时保留结构日期与可选事件；节气或节日不得覆盖换月信息。空间不足时，低优先级内容折叠为数量提示，通过悬浮、焦点或详情弹层查看；普通月格只保留进一步压缩后的 `52px` 通用基础最小高度，不为尚未出现的节假日或 ICS 预留空间。地区节假日名称与 ICS 均按实际可见行数占用高度；可见内容增多时只让所属周七个日期格与周序号统一自然增高，不影响同月其他周。月首尾的非本月日期使用更淡的整格底色、边界和内容表达；今日使用贴格子内缘的中性 `1px` 内边框，选择使用位于格子边界之外的强调色 `2px` 外框，焦点再在 selected 外侧作为独立最外层显示，今日不得复用笔记状态圆点。current、selected 和 focus 均不改变背景、文字颜色、字重或装饰。当日、选择、区间预览、悬停及键盘焦点必须恢复完整对比度，焦点轮廓不得随内容透明度一起弱化。

左侧周序号是周记入口，不是静态标签。行内使用紧凑的 `W` 前缀与等宽周号（如 `W27`），列头仍按界面语言显示“周/週/W”；月主网格默认使用 `6px` gap，`≤360px` 时收紧为 `4px`，周索引轨道与七列日期之间另有稳定 spacer 留白，不使用贯穿分隔线，也不得伪装成第八个日期格。每个周序号入口作为独立格，四角均采用与日期格一致的 `5px` 圆角，不沿用连续轨道遗留的右侧直角。单击选择整周；双击、Enter 或长按默认打开周记；中键或 `Ctrl / Cmd` 单击在新标签打开。周记没有地区标记，因此状态圆点或横向任务进度使用顶部绝对叠加层，`W` 标签始终相对整个控件垂直居中。包含本地今日的周序号使用 `is-current-period`、贴格子内缘的中性 `1px` 内边框和 `aria-current="true"`，所选周增加位于格子边界之外的强调色 `2px` 外框；两者可同时存在，焦点在 selected 外侧作为独立最外层显示，且均不改变背景、文字颜色、字重或装饰。周记关闭、月热力图或“显示笔记状态”关闭时完全不挂载状态叠加层，也不保留空状态行；current/selected 状态、周记路径和完整无障碍文本仍保留。窄侧栏必须容纳 `W53` 与相应指示器。

ICS 在普通宽度的日期格中最多显示三条稳定排序摘要：起始日显示全天或本地开始时间，跨日段用前后省略号表示延续，更多事件显示 `+N`。完整事件标题、来源、全天/时间和前后延续状态全部进入日期格无障碍名称，包括折叠事件。月热力图、窄容器和粗指针布局可隐藏可见摘要，但不得改变上述无障碍信息。

中国农历在中文界面使用传统月日文本；英文界面使用紧凑的 `Lunar M/D` 数字文本并显式标注闰月，节气与传统节日使用英文名称。干支历在中文界面显示中文干支、英文界面显示拼音干支，平日显示日柱，交节日显示新的月柱。完整的本地化历法文本进入日期格无障碍名称，扩展自身 tooltip 保留完整年/月/日柱或农历月日。历法只提供日历信息，不提供命理内容。
地区标记固定在顶部右侧，并跟随插件解析后的界面语言：简体/繁体中文使用 `班 / 休 / 公假`，英文使用 `Work / Off / PH`。颜色只能作为辅助表达。节假日地区使用“地区 1 / 2 / 3”三个中性有序槽位；补班标记始终优先，其余同日标记按槽位顺序选择，DOM 中只能出现一个可见地区标记。假日名称位于历法扩展下方、ICS 上方，多个地区按设置顺序分行、同地区名称在行内排列并居中；未赢得顶部标记的假日仍须在无障碍名称和悬浮/焦点预览中保留完整内容。

悬浮或键盘焦点停留 250ms 后显示只读笔记预览；离开或失焦立即取消。预览必须避开当前日期格、限制在视口内，并在滚动或缩放时重新定位。空文件、仅 YAML、无可预览正文和读取错误都显示明确文本；关闭悬浮预览设置时必须立即取消待触发 timer 和已挂载 tooltip，且不得创建旁路读取或后台请求。年度每日预览只属于热力模式；切到概览、切换显示年份或关闭预览都必须同时取消 timer、卸载 tooltip 和清除 ARIA 关联，旧预览不能在新模式短暂闪现。

## 4. 笔记状态

- 索引中：弱化色虚线空心形状；
- 不存在：错误色空心形状；
- 空文件：强调色空心形状；
- 仅 YAML：强调色实心形状；
- 有正文：成功色实心形状；
- 有任务：使用任务完成进度替代普通圆点；
- 读取失败：警告图标和可读错误提示。

Tooltip 必须说明状态含义，不能要求用户记忆颜色。

路径仍未知时，日期右键菜单不显示笔记打开/创建动作；双击、长按、Enter、Ctrl/Cmd 单击、Properties 日期入口和命令面板等直接周期笔记命令显示本地化的“索引仍在更新”提示，不得创建文件。已有 parsed/error entry 仍可按原语义打开。

## 5. 交互

主日历统一用 `is-current-period` 表达包含本地今日的精确日期或周期，并把 current、selected 与键盘 focus 作为正交状态。日期、周序号以及年概览的月份/季度入口都以贴格子内缘的中性 `1px` 内边框表达 current；selected 使用位于格子边界之外的强调色 `2px` 外框，键盘 focus 再位于 selected 外侧作为独立最外层。三种状态可以同时存在，均不改变背景、文字颜色、字重或装饰。格间 gap 必须为 selected 外框预留真实空间，不能以 inset 或向内缩进伪装外框。精确日期使用 `aria-current="date"`，周、月、季周期按钮使用 `aria-current="true"`，选择继续使用对应控件的 `aria-selected` 或 `aria-pressed`。月区间预览/端点和周任务拖放目标等临时态优先，但 selected 与 focus 仍保持独立。月/年热力图继续保留数据等级背景，并复用同一 current/selected/focus 框线矩阵。Calendar、Note Navbar 和区间列表中的原生 button/select 在宿主主题下都必须显示清楚的 `:focus-visible` 外框；日期格继续使用自己的最外层焦点环，不叠加双框。该主网格契约不直接扩展到周期选择器或迷你日历。周期选择器独立使用强调文字色和 `aria-current` 表达 current，不使用圆点；selected 只增加 `2px` 内描边和 `aria-pressed`，不改变文字色或背景；focus 使用独立外描边，current 与 selected 可以共存。迷你日历保持既有状态样式。

- 单击：选择日期或周期；
- 双击/长按：打开或创建对应笔记；
- Ctrl/Cmd 单击或中键：在新标签页打开；
- 右键：打开上下文菜单；
- 方向键：移动选择；
- Enter：打开；
- Escape：关闭弹层或取消范围选择。

从日历打开或创建周期笔记时，必须先把对应日期或周期设为语义选择，中键、键盘和长按路径同样适用。通过 Obsidian 任意入口激活一个已启用且路径可识别的周期笔记 Markdown 时，所有已打开的 Chrono Notes 日历都导航到该周期，但保留各自当前的周、月或年视图；普通或无法识别的 Markdown 不改变日历。日记选择精确日期，周记选择周行或周记入口，月记选择月份，季记选择季度，年记选择年份入口。年热力图选择精确日期格；年摘要保持摘要模式，展开并滚动到所属季度，选择所属月份，并在月份内显示精确日号或 `Wxx`，不得强制切换热力图。周选择用一周中间的参考日决定显示月份和周年，因此跨年周显示在多数日期所属月份，同时保持规范周记路径不变。

触摸长按使用 500ms 阈值打开对应笔记；touch move、touch end 或 touch cancel 在阈值前发生时必须取消。长按成功后只抑制一次浏览器合成 click，不能再次切换选择或重复打开；组件卸载必须清理未完成计时器。

桌面月视图用无修饰键左键按下建立拖选锚点，进入另一个日期格时显示规范化范围预览，反向拖动同样有效。只有跨过至少两个不同日期并以左键结束才提交创建；单格按下/抬起仍执行普通选择，完成拖选后吞掉随后一次 click，Escape 或失去有效拖动状态只清理预览。创建必须走现有区间命令和确认链，视图不得直接写 Vault。

日期右键菜单中的“创建区间笔记”与日记是否启用相互独立，只在区间创建目录非空时显示。输入弹窗默认以所选日期为开始、次日为结束，只接受两个不同的 `YYYY-MM-DD` 日期并允许反向输入。区间确认开启时，继续后必须显示规范化范围、包含首尾的天数和目标路径，再由用户确认创建；确认关闭时只跳过这一步，日期输入仍保留。已有同路径笔记直接打开，不重复确认或覆盖。确认弹窗中的“以后创建区间笔记时不再确认”默认不选中；只有选中并确认创建才关闭该类别的后续确认，取消、关闭或 Escape 不修改设置。

普通月视图和周视图使用同一个简化甘特组件；月热力图不渲染区间甘特。区间横条覆盖开始到结束日期列，不再显示为独立日期范围格；重叠区间进入下一泳道，不重叠区间复用泳道。普通月视图在完整的 4–6 周可见窗口内一次分配泳道，使同一区间跨周片段保持相同纵向位置与稳定身份色。月视图的每个周块使用日期内容轨与区间轨组成的双轨 grid；周序号、留白和七个日期按钮的背景与边框跨越两轨，日期信息只位于第一轨。区间甘特在第二轨正常参与布局，位于 ICS 之后、日期按钮下边框之前。无区间时第二轨折叠为 `0`；多条泳道只撑高所属周，不得迫使整个月的其他周采用同一高度。实现不得使用绝对 overlay、估算的底部留白或平行高度函数模拟区间空间。精细指针下，每个可见月泳道是独立且互不重叠的 `10px` 交互行，`4px` 视觉线在行内垂直居中，泳道间距为 `0`，strip 底部留出 `2px`；粗指针下只把交互行扩为 `18px`，视觉线仍为 `4px`。不存在另加的透明命中层，相邻泳道不得共享命中区域，ICS 也不得与线条重叠。月/周视图的可交互区间条都必须是日按钮之外的独立按钮；月甘特容器是日按钮所在行的 sibling，周甘特则位于七个日按钮之后的正常流 sibling 区域，均不得嵌套交互控件或重复列出区间。月视图隐藏条内标题，以颜色、tooltip 和 ARIA 保留身份；周视图空间充足时保留条内标题。边缘省略表示继续。颜色来自八个分类色，并向当前主题前景色校正明度，使静止状态下也与日期格背景清楚区分；hover/focus 必须清晰但只提供附加反馈。同一规范笔记路径在月视图中跨周产生的所有可见片段共享联动态：鼠标悬浮或键盘聚焦任一片段时，其他同路径片段同时高亮，触发片段的反馈更强；移出或失焦后清除对应联动态。关联只能按规范路径判断，禁止按相同分类颜色或泳道号判断；联动与原片段反馈只改变颜色、描边或层叠，不改变行高、间距、线宽、位置、列跨度或命中区域。颜色只辅助区分身份，不使用成功/警告/错误语义，也不能替代月视图的 tooltip/ARIA 或周视图的条内标题。超过泳道上限时显示 `+N`，hover/focus 可读取隐藏标题。完整 tooltip 包含标题、原始日期/时间范围和包含首尾的天数；单击按旧语义在新标签打开。

周视图与月/年视图使用同一分段控件。顶部以独立周历年和周号入口取代两行标题与迷你日历：周历年入口复用 20 年窗口，周号入口按时间顺序列出该周年全部有效周。周弹层理想宽度为 `360–420px`；可用宽度至少 `360px` 时每行三周，更窄时每行两周。每周是单一 `44px` 原生按钮/焦点目标，内部为 `W01 | 12/29–1/4` 两列；同月范围压缩为 `7/20–26`，完整年份和起止日期保留在 tooltip/ARIA。弹层限制高度并纵向滚动，打开时把所选周滚入视野。年、月、ISO 周年和周号弹层统一以强调文字色加 `aria-current` 表达 current，不使用圆点或字重变化；selected 只使用 `2px` 内描边和 `aria-pressed`，不改变文字色或背景；focus 使用独立外描边，current 与 selected 可同时存在。月份仅在弹层所浏览年份等于本年时标为 current，周号仅在所浏览 ISO 周年匹配当前周年时标为 current。方向键按视觉网格移动，Home/End 到首尾，Page Up/Down 跨五行，连续输入一至两位数字快速定位周号。前后按钮按 `±7` 天导航并保留所选星期。七日概览复用月格的完整 `CalendarDay` 信息层级：第一行是状态/横向任务进度与调休/地区标记，第二行是星期，然后依次为日期、历法扩展、节假日和 ICS。周主网格默认使用 `6px` gap，`≤360px` 时收紧为 `4px`；周区间甘特使用相同的 `6px / 4px` 列间距与七日格对齐，日期与其后详情保持 `3px` 有效间距。概览对辅助技术表达为带名称的原生按钮选择组，每日按钮以 `aria-pressed` 表达所选日期，今日则独立使用 `aria-current="date"`；由于没有行结构和 roving focus，它不得标成 ARIA grid。宽档显示完整星期与最多三条 ICS，中档显示短星期和一条 ICS，窄档显示最窄星期/日期并收起详细内容；三档都保留完整 tooltip/ARIA。今日使用贴格子内缘的中性 `1px` 内边框，选择使用位于格子边界之外的强调色 `2px` 外框，焦点再在 selected 外侧作为独立最外层显示；三者均不改变背景、文字颜色、字重或装饰，任务拖放目标等临时态优先。单击选择，双击/Enter 默认打开，中键或 Ctrl/Cmd 在新标签打开；已配置的周记以单独紧凑行提供打开/创建入口。无区间时不显示“暂无区间”段落，仅在创建目录有效时保留以周首日为起点的紧凑创建按钮；无任务时标题中的零计数就是空状态，不再重复文案。

周任务列表聚合整个 Vault 中带 due、scheduled 或 start 日期的任务，同一任务同日只显示一次，并标明触发日期类型。未完成任务只按 due date 表达逾期：1–2 天为警告，3 天及以上为严重；完成任务和仅 scheduled/start 的任务不标逾期。逾期行必须显示“已逾期”或“严重逾期”等等级文字，并在复选框无障碍名称中同时保留日期类型和等级；颜色仅作辅助，窄侧栏可以隐藏普通日期类型，但不能隐藏逾期等级。checkbox 通过命令层切换，任务文本在新标签打开并定位来源行。只有显示在自身 due 日期分组中的任务实例可改期：桌面可拖拽到当前周的另一天，任务行同时提供本周七日原生下拉框供键盘与触摸操作，并使用本地化日期及完整任务名作为标签。scheduled/start 重复实例既不可拖也不显示该下拉框；两条路径都等待 Vault 事件刷新，React 不直接改文件。

区间笔记列表可从命令面板和区间设置页打开。搜索同时匹配标题与路径且不区分大小写；范围可选全部、与当前月相交或与当前年相交；排序可按开始日期升序或降序，并以结束日期、标题和路径稳定打破并列。工具栏显示结果数并提供创建入口，空索引与无匹配结果使用不同状态；单击条目在新标签打开。列表在 Vault create/modify/rename/delete 和扫描设置变化后即时刷新，窄屏工具栏改为纵向排列且不截断路径。

首次启用插件且工作区布局就绪后显示一次简短引导。为避免重载或多个布局回调重复打扰，打开 Modal 前先持久化“已看过”标记；关闭按钮、打开设置或 Escape 均视为已看过。常规设置页始终提供“重新打开”入口，手动重开不重置标记。

引导只说明当前真实可用且容易错过的交互：单击只选择日期/周期，双击或 Enter 默认打开，中键或 Ctrl/Cmd 单击在新标签打开，触摸端可长按；日期右键提供笔记、区间和复制动作；日期格的形状/数字/文字表达笔记、任务与日历状态；周期笔记的 Note Navbar 可前后和向上导航。引导使用原生 Obsidian Modal，提供“打开设置”和“知道了”两个操作，不制作营销页或多步遮罩导览。

可点击目标需要焦点样式和适当的触摸尺寸；Calendar、Note Navbar 与区间列表的 button/select 不能依赖可能被宿主清除的默认 box-shadow。周任务改期下拉框在粗指针下至少 `44px` 高，窄侧栏允许收窄但不得截断其可访问名称。

## 6. 设置页

外观与视图中的节假日地区使用“地区 1 / 2 / 3”三个中性有序下拉槽位，只列出当前已实现 provider，排除其他槽已选项，并在清空前槽时把后项自动前移。

五个顶部选项卡：常规、外观与视图、周期笔记、区间笔记、集成。窄屏不换成多行，而是保持单行横向滚动；点击、键盘激活或设置页重绘后，活动选项卡必须自动滚入可视区。标签栏隐藏纵向溢出，不能出现细竖向滚动条；桌面精细指针下标签高度为 34px，粗指针下为 44px。

语言设置提供自动、English、简体中文、繁体中文、阿拉伯语、波斯语、希伯来语、阿姆哈拉语和印地语。切换后已打开界面立即刷新；产品文案、tooltip 和 ARIA 使用同一完整消息目录契约，八套目录必须覆盖同一键集合及相同插值占位符，日期、月份、星期与数字使用解析后的 locale 交给 `Intl`。阿拉伯语、波斯语和希伯来语把设置页与日历根切换为 RTL，其他语言为 LTR；同一信息层级在两个方向中保持语义顺序、可点击尺寸和对齐，空间变化不得造成按钮溢出或控件跳位。

外观与视图页提供字体模式（跟随 Obsidian、跟随侧栏、固定倍率）、两个有序历法扩展槽位、“显示笔记状态”开关、季度命名（本地化数字名称、`春 / 夏 / 秋 / 冬`）和任务标注（不标注、颜色标注、空心标注）。每个历法槽使用下拉选择且不提供另一个槽已选的扩展；清空第一槽时第二槽自动前移。固定模式提供 `0–20` slider，默认 `10`；非固定模式保留该值但禁用 slider。跟随侧栏只能用离散容器档位改变字号，不能随视口宽度连续缩放。任务标注默认使用空心模式：无任务继续显示笔记状态，全部完成统一使用成功色；未完成任务在颜色模式使用警示色实轨，在空心模式使用警示色空心轨道。不标注只把可见任务进度回退为笔记状态符号，不得删除任务统计、热力图/预览信息、任务列表或 ARIA。季度命名切换必须立即同步年概览、年热力图季度标题和月选择器，不能改变季度锚点、笔记路径或 Note Navbar 标签。显示开关开启时，月、周、年统一使用顶部状态圆点或横向比例进度；完成数与总数保留在 ARIA、tooltip 和数据属性中，不以分数文字挤占格内空间。关闭开关不挂载可见圆点/进度，但不删除笔记与任务状态。常规页提供“使用 Chrono Notes 打开日期属性”开关，默认关闭；只在日记已配置时捕获日期属性值旁的打开图标。普通点击沿用默认打开目标，`Ctrl / Cmd` 点击在新标签页打开；Properties 中的 Wiki/Markdown 链接、正文链接和其他控件全部保持 Obsidian 原有行为。

周期笔记页把五类笔记作为独立折叠段落：关闭时只显示类型与启用开关，启用后显示路径格式和模板路径。行为区提供独立的“创建周期笔记前确认”开关；确认弹窗中的“以后创建周期笔记时不再确认”默认不选中，只有选中并确认创建才关闭后续确认，取消、关闭或 Escape 不修改设置，较大周期级联不重复弹窗。五类段落前始终显示共享格式说明，明确路径格式同时包含文件夹与文件名、英文字母等固定文本使用 Luxon 单引号而纯中文文本可直接写、Vault 根目录写法，以及 Obsidian `YYYY-MM-DD` 到 Luxon `yyyy-MM-dd` 的转换。每个路径输入使用包含文件夹的本类型完整示例作为 placeholder，并把该示例放在始终可见的独立一行。路径格式与模板路径控件无论面板宽度如何都固定使用单列：标题和说明在上，输入框在下一行占满可用宽度。有效路径在输入框正下方用独立一行显示“当前预览”；空值、常见 Moment token 和无法通过统一反向识别的格式则在同一位置显示针对性校验错误。错误状态使用错误色、`aria-invalid` 和可朗读动态说明。Obsidian 原生键盘候选列出 Vault 文件夹，锚定的建议弹层与全宽输入框等宽；选择文件夹时保留当前文件名格式，空格式使用该周期的规范默认文件名格式。预览使用当前日期和周起始规则生成完整 `.md` 路径。模板输入按周期显示不同 placeholder，只建议 Vault 中的 Markdown 文件并写入完整相对路径，留空仍表示创建空白笔记；字段名称不与当前模板引擎耦合。手动输入始终可用，路径输入关闭自动大写、自动完成与拼写检查，候选与预览不得建立第二份配置或读取笔记正文。

区间笔记页提供独立的“创建区间笔记前确认”开关、日历显示开关、创建目录、扫描范围（区间目录/自定义目录/整个 Vault）、自定义扫描目录和月/周显示上限。仅在选择自定义范围时启用自定义目录输入；目录为空时禁用创建入口，但不伪造或删除既有区间数据。

集成页提供本地 ICS 显示开关、每行一个只读来源、手动刷新和逐来源状态。来源可为 Vault 文件、桌面 Vault 相对路径或桌面绝对路径；读取失败与成功来源同时展示，刷新不得清除仍可用来源的事件。状态必须明确列出事件数、跳过的 recurrence 数和错误，不能把 recurrence 静默当成已导入。

选项卡使用 `tablist/tab/tabpanel` 语义，支持左右方向键、Home 和 End。依赖条件不满足的设置应禁用并解释原因，而不是无提示隐藏。

## 7. 响应式与性能

- 根据容器宽度而不是窗口宽度调整密度。
- 年视图在概览模式中按季度显示季度与三个月入口，格间距常规为 `6px`、`≤360px` 时为 `4px`；“本月”会切换到当前年、以 `day: 1` 月锚点选择当前月，并挂载及滚入对应季度。可见的季度/月笔记状态使用距格顶 `4px` 的绝对叠加层，与文字保留约 `2px` 可见间距；周期标签始终相对整个按钮居中，入口格高保持不变，笔记类型关闭或状态隐藏时不挂载空状态槽。包含本地今日的季度/月入口使用 `is-current-period`、贴格子内缘的中性 `1px` 内边框和 `aria-current="true"`，选择增加位于格子边界之外的强调色 `2px` 外框，焦点再位于 selected 外侧作为独立最外层；三者均不改变背景、文字颜色、字重或装饰。热力图中不显示状态的周期按钮遵循同一居中和 current/selected/focus 规则。热力图模式按季度显示三个月的全年每日强度格，日热力小格间距常规为 `4px`、`≤360px` 时为 `3px`，并提供与月视图共享的统计维度和 0–4 级图例。
- 月、季和日均可选择；双击或 Enter 默认打开，中键或 Ctrl/Cmd 在新标签打开。年度每日格保留右键菜单、仅热力模式启用的悬浮/焦点预览、方向键和非颜色 ARIA 值；每个月 grid 必须包含七个实际 `row`，每日 `gridcell` 归属对应行。由于日期按星期行、周列排列，Left/Right 分别移动 `-7/+7` 天，Up/Down 分别移动 `-1/+1` 天，与视觉方向一致。今日使用 `aria-current="date"`，保留热力等级背景并叠加贴格子内缘的中性 `1px` 内边框，选择使用位于格子边界之外的强调色 `2px` 外框，焦点再位于 selected 外侧作为独立最外层。所选日属于当前显示年且已渲染时只有该日期可由 Tab 到达；尚无本年已渲染日级选择时以 1 月 1 日作为唯一临时 fallback，目标季度挂载后停靠点必须迁移到所选日，跨显示年的旧选择不得让网格失去停靠点。
- 年视图只渲染当前季度窗口及相邻 overscan，未渲染季度使用与当前响应式实块等高的稳定占位。切回月视图时保留最近选择的月份上下文。
- 月视图避免每个日期格独立读取文件。

区间甘特超过泳道上限时，`+N` 必须是带 `aria-expanded`、`aria-controls` 和 `aria-haspopup="dialog"` 的真实按钮，而不是只带 tooltip 的静态文本。它打开仅包含当前周隐藏区间的面板；每项保留规范路径、标题、原始起止边界、包含首尾的天数和任务统计，使用至少 `44px` 的独立按钮目标，单击仍在新标签打开。桌面面板锚定当前甘特，移动端限制在系统安全区内；Escape 或点按外部关闭，Escape 后焦点回到触发按钮。
