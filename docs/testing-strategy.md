# Testing strategy

Testing is layered: pure core unit tests, port contracts, feature integrations, focused UI behavior, and manual Obsidian acceptance. Prefer durable invariants over broad snapshot coverage.

Critical invariants include equivalent BOM/line-ending/frontmatter inputs producing the same state, stale asynchronous work never overwriting a newer revision, complete Vault event convergence, shared month/year statistics semantics, reversible periodic path rules, and independent mutable setting arrays and nested configuration objects.

Every bug fix starts with a reproduction. Coverage reveals gaps but does not replace contract tests.

NoteIndex port and integration tests pin the following failure model. Deduplicated initial paths reserve revisions up front and run under the default 16-worker bound; their separate staging area merges atomically and publishes once only after every read succeeds or fails explicitly. Live events during the scan advance path revisions immediately, but ordinary create/modify operations publish through microtask intent and completion batches without exposing staged initial results. Delete and rename first form an immediate old-path invalidation barrier. Contracts cover concurrent-refresh reuse within one revision, create→modify, create→delete, delete→recreate, A→B→C, out-of-order completion across revisions, a slow file not blocking unrelated changes, late deletion results, discarded pending work across stop/restart lifecycles, extension-changing renames, `refresh()` joining the correct promise, and preservation of older snapshots. A subscription, path-listing, or internal-initialization exception must remove the subscription and every half-started state, after which the same instance can start successfully. A modify with the same canonical document performs one read/document parse but skips full domain derivation and public publication; error recovery and rename force full parsing. A high-cardinality projection regression pins last-change-wins per path, complete task-bucket and interval ordering, at most one sub-snapshot revision per batch, identity retention for a no-op batch, and use of the same batch boundary by initial and live commits. Adapter tests pin the `cachedRead()` boundary with a structural fake Vault without introducing the Obsidian runtime into core.

Readiness regressions additionally pin `indexing` before initial completion, `ready` after the atomic startup boundary, a live unknown-path wave returning to `indexing`, and restoration to `ready` only after the injected quiet checkpoint sees no pending intents, reads, queues, or batches. Calendar-store tests prove that a readiness-only snapshot recomputes unknown visible periodic paths from missing to indexing without rematerializing notes, while parsed/error entries retain their meaning. Presentation tests distinguish the muted dashed indexing marker from the error-color missing marker, and command acceptance must prove that an unknown path cannot create a periodic note.

Integration regressions combine the real `ObsidianNoteSource` with a controllable fake Vault and complete `cachedRead()` calls out of order across initial→modify, pending modify→rename, and pending modify→delete. They prove that an old read cannot overwrite a newer publication or revive an invalidated path. This pins adapter event mapping and the NoteIndex revision barrier, not real Obsidian cache invalidation after external synchronization; rapid desktop/mobile edits and sync remain a manual release gate. When `refresh()` encounters a queued missing intent, it commits the deletion barrier first and then performs an explicit read under a newer revision; tests pin listeners observing missing before that read's parsed or error result becomes authoritative.

Settings contracts cover sequential Chrono Notes schema v1–v15 migration, treating missing and invalid versions as v1, leaving future versions undowngraded, preserving valid values, dropping unknown fields during final normalization, migration idempotence, and independent input, default-array, and nested-object references. Schema v15 proves the shared confirmation field is removed without translating its value and that the periodic and range strict-boolean toggles normalize independently. Manual Obsidian acceptance injects representative older and invalid data into the isolated vault, performs a full reload, verifies preservation and fallback across all five settings tabs, then saves the current schema and proves that array settings do not alias each other.

Periodic command tests cover unconfigured and existing notes, creation confirmation, one anchored template context, parent-folder creation, target-date-aware built-in templates, rollback after a main template failure, cascade skip/failure/continuation, and preservation of a successfully written file when view opening fails. A same-path concurrency regression keeps creation and templating to one execution while proving the creator/waiter `created` results, independent open targets and cascades, and retry after failure. Shared confirmation-orchestration contracts prove cancellation does not save, ordinary confirmation does not save, only confirmed suppression persists, and cascades do not prompt again. Structural Vault and Workspace tests cover adapter boundaries; manual Obsidian acceptance still covers real modals, the command palette, and themes.

Periodic-settings presentation contracts cover empty values, formats that fail reverse recognition, targeted correction of Obsidian's `YYYY-MM-DD`, Luxon single quotes for fixed Latin-letter and similar token-like literals, direct round trips for unquoted pure-Chinese folders, all five complete path and template examples, nested folders, Sunday-start ISO cross-year previews, Vault path separators, and Luxon escaping for apostrophes in folder names. A source boundary pins dependent controls collapsing before render, native Obsidian keyboard suggestions for folders and templates, accessible `aria-live` / `aria-invalid` validation with auto-capitalization disabled, and `getMarkdownFiles()` as the sole template-candidate source. Manual Obsidian acceptance adds mouse and keyboard selection, complete relative-path write-back, live previews and errors, save/reload, all three UI languages, and a narrow settings panel.

Labels and recursive freezing remain fixed. Layout boundary tests pin the symmetric three-column grid, peer and higher-level navigation in one centered middle group, open-calendar alone at the trailing edge, and narrow containers hiding only higher-action text while retaining the button. Settings tests pin current-schema-v15 normalization for visibility, collapse, first-use-marker booleans, ICS source arrays, both creation-confirmation toggles, `showNoteIndicators`, quarter naming, font mode/factor, task annotation, and Properties date interception, and prove removed fields do not determine new toggles. Manual Obsidian acceptance covers all four entry points: main title, Navbar, mini-calendar command, and direct-jump command. A cancelled recurring event does not contribute to the recurring count. Feature-index tests cover source normalization and deduplication, partial failure, same-source reuse within one request revision, an immediate fresh same-source read across revisions, disable-then-enable, out-of-order configuration revisions, late results after disable/stop, and notifications, including invalidation after an old read finishes but before expensive parsing. Presentation contracts pin at most three summaries in a regular month cell plus narrow/coarse folding, and at most three, one, and zero summaries in wide, medium, and narrow week cells, with the correct `+N` for each visible count. They also prove all-day/time/continuation data and every folded title and source remain in ARIA. CSS boundaries pin content-driven ICS auto rows, no fixed blank height when events are absent, and visible-only folding in narrow containers and coarse-pointer layouts. Obsidian port tests cover indexed Vault files, adapter-readable Vault files before cache readiness, desktop absolute and relative paths, CommonJS desktop-module loading, and explicit errors for nonfilesystem vaults; production bundles must not retain browser `node:` dynamic imports. Manual acceptance adds real paths, permission errors, settings status, repeated refresh, cross-day summaries, `+N`, narrow sidebars, light/dark themes, and desktop/mobile differences.

`IcsEventIndex.stop()` contracts additionally prove idempotence, release of the complete event mapping without listener notification, and rejection of a pending refresh result after that release.

Test counts continue to grow, so the authoritative final count is the output of the corresponding `pnpm check`, not a hard-coded value in this stable document.
- Architecture tests use the TypeScript AST to pin layer direction, core independence from Obsidian and React, and absence of source cycles.
