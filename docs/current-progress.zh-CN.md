# 当前状态

## 实现状态

- 旧项目功能范围已经在当前实现中闭合，具体能力见[功能对等清单](feature-parity.zh-CN.md)。
- NoteIndex 是 Vault 笔记事实的唯一所有者；任务日期、区间和日历查询使用增量投影与结构共享。
- 月、周、年三种日历视图、五类周期笔记、任务、区间、八种界面语言、八个历法扩展、地区节假日和本地 ICS 均已接入当前数据主干。
- 设置页使用原生 Obsidian API；路径候选共享可释放目录并限制为排名最高的 100 项。
- 月/周区间溢出通过可访问的 `+N` 按钮打开当前周隐藏项，条目保持新标签打开语义。
- 插件先完成 UI 与宿主入口注册，布局就绪后后台启动 NoteIndex 和首次 ICS 刷新；索引 readiness 继续阻止未知路径被误建。
- 周期笔记删除与创建失败回滚使用 Obsidian 回收站接口；英文历法和中国大陆节假日不再残留中文名称。
- 日历内的打开/创建动作与 Obsidian 活动周期笔记使用同一语义选择；周/月/年视图保持不变，年摘要以月份内日号或周号表达精确选择，跨年周按多数日期所属月份导航。

## 当前自动验证

- `pnpm check` 通过，包括源码格式、严格类型、完整 Vitest、生产构建和产物 smoke/size 契约。
- UTC 与 `America/New_York` 时区测试通过。
- 1,000 与 10,000 篇确定性基准通过；live create/modify 批次保持每终态路径一次读取，快速 32 路批次一次发布。
- 冷月装饰基准保持在十几毫秒量级，不再重复触发同语言的 lunar 全局消息更新。
- production `main.js` 低于 1,150,000 B 门禁；该预算以已发布 0.1.2 产物为基线，覆盖五套完整新语言目录的可归因增量并保留约 50 KB 余量。明细输出预算余量、相对基线增量和依赖包聚合，精确大小以当前 `pnpm release:check` 输出为准。
- 年度节假日门禁通过；中国大陆下一年仅在一手来源确认官方安排尚未发布时以带来源警告的 `unavailable` 状态继续。
- 当前 0.2.0 production 产物已在自动生成的隔离桌面 Vault 中复验英文与阿拉伯语设置、LTR/RTL 月视图、窄侧栏、ICS、日记状态、任务进度与班休标记共存；六个新增 `Intl` 历法均由实际 Electron 宿主确认可用。
- 同一 production 文件已按 SHA-256 部署到 Android 15 / API 35 模拟器的自动生成夹具副本；月/周/年、阿拉伯语 RTL、横竖屏、周期选择器返回、前后台恢复、区间/任务和六个新增历法均通过，日志无插件异常、崩溃或 ANR。尚无物理 Android 设备证据。

## 当前发布状态

当前产物必须通过完整自动门禁和与本次变更相关的真实宿主回归。完整桌面兼容矩阵、真实移动设备矩阵和真实宿主大 Vault Profiler 按变更风险触发，而不是要求每个补丁版本无差别重跑：

1. 最低支持 Obsidian 1.12.7、执行时最新稳定桌面版或宿主集成表面发生变化时，重跑对应桌面矩阵；
2. 安全区、周期 Note Navbar、触摸、横竖屏或软键盘行为发生变化时，补真实移动设备矩阵；
3. NoteIndex、查询、缓存、依赖或产物预算发生性能相关变化时，补 1,000/10,000 篇真实宿主 Profiler、最长主线程任务和 heap 证据。

未触发或未取得的证据只限制对应兼容性、真机或性能结论，不阻止无关补丁发布，也不得被写成已经验证。

完整执行口径见[测试策略](testing-strategy.zh-CN.md#人工发布门禁)。
